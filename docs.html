Everything you need to integrate Error AI into your applications, bots, or research tools.

Authentication

All requests to Error AI require an API key. You can generate one via our Telegram Bot.

Include your key in the Authorization header for every request using the Bearer scheme.

Authorization: Bearer errorai-your_generated_key_here
Note: Keep your key safe! If it's compromised, use /revoke in the Telegram bot immediately.

Chat Completions

The main endpoint is designed to be a drop-in replacement for OpenAI. It supports real-time streaming and custom system instructions.

POST /v1/chat/completions 
curl https://errorapi.dev/v1/chat/completions \
  -H "Authorization: Bearer errorai-YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "dolphin-llama3",
    "messages": [{"role": "user", "content": "How does thermodynamics work?"}],
    "stream": true
  }'

Parameters:

  • model: Currently supports dolphin-llama3.
  • messages: Array of message objects (role & content).
  • stream: Boolean. If true, tokens are sent via Server-Sent Events (SSE).

Adaptive Persona

You can dynamically update the AI's "identity" or background context. This persists across all requests made with the specific API key.

POST /api/v1/persona 
{
  "persona_context": "The AI is an expert researcher in dark energy and cosmic inflation."
}

Rate Limits

Each key has both a daily limit and a total lifetime usage limit. These are controlled live by the Error Community via the Telegram management bot.

Default Limit: 100 requests / day.
