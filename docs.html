API Documentation

Integration guide for Error AI endpoints.

Getting Started

To use the Error AI, you must first generate an API Key using our official Telegram Bot. Your key acts as a Bearer token for all requests.

Base URL: https://example.com

Authentication

All API requests must include the Authorization header with your Bearer token.

Authorization: Bearer railway-YOUR_API_KEY

Chat Completions

Our main endpoint is fully OpenAI compatible. You can use it with any standard library (OpenAI Python, LangChain, etc.).

POST /v1/chat/completions

Example Request (cURL):

curl https://example.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_KEY" \
  -d '{
    "model": "dolphin-llama3",
    "messages": [{"role": "user", "content": "Hello, Errro!"}],
    "stream": false
  }'

Adaptive Persona

Update your AI's speaking style and context dynamically. This will affect all subsequent chat completions for the provided key.

POST /api/v1/persona 
{
  "persona_context": "The user is a sarcastic hacker who loves 1990s cyberpunk terminology."
}
